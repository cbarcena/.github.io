Cascading Style Sheet

Cascading Style Sheets (CSS) is a presentational language developed to control the presentation of structured documents such as fonts.

HTML has Evolved into a primarily structured language Previously presentational markups such as <font>, <b>, <i> have either been deprecated or redefined within the language to reduce the presentational nature of elements.

CSS is a collection of formatted rules usually external. There are three ways CSS can be implemented on a website.

  1. Cascading external CSS document: main.css
  2. Embedded HTML Styles: <style> </style>
  3. Inline styles: <p style=” ”></p>

Based on where the styles are found within the <html> the styles are allowed to override one another when conflicts arise.

What makes up CSS?

CSS is Made up of 2 parts, a Selector and a Declaration. A Selector is the element, class or id you are trying to style. A Declaration is enclosed in curly braces w/ formatting instructions. Each declaration is made up of 2 parts, a property and a value.

Benefits of CSS

CSS documents have created a way for designers to generate portable designs that can be reused or changed when necessary. All without the hassel of needing to make the same change on multiple pages for similar elements. By including CSS in your code you can make one single adjustment that will take effect on each page

  1. Allows for modular site design
  2. Faster development by the centralization of styles

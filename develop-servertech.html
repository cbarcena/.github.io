Choosing a Server Side Language:

What to consider:

  • Cost and developer network availability ease of use
  • Find a web developer that can assist in the right platform
  • With there being many different server languages learn as much as you can about various platforms to make an informed decision

Common server languages:

  • JSP: Java Server Pages
  • APS.net: Visual Basic .Net
  • CFML: ColdFusion
  • Python
  • Ruby

Working With Clients and Servers

  • URL (Uniform Resource Locator) When a request is made DNS (Domain Name Service) translates the URL to an IP address, browser uses the IP address to locate host server and send request for content
  • Depending on the type of information requested, such as a simple HTML page, the HTML text is rendered and presented to the user.
  • If additional information is requested, such as CSS or images, these requests are presented to the server, resources and are rendered then presented.
  • Client Side Scripts - JavaScripts are executed at this point
  • Static Sites - No additional processing, just the content requested by the client
  • Dynamic Sites - uses business logic, processed by web application(s), such as PHP, .NET or Ruby before served to the browser.
  • Requests must be made, sent to the server then, sent back to client rendered then processed
